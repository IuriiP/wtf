Class ParserTest

A TestCase defines the fixture to run multiple tests.

To define a TestCase 1) Implement a subclass of PHPUnit_Framework_TestCase. 2) Define instance variables that store the state of the fixture. 3) Initialize the fixture state by overriding setUp(). 4) Clean-up after a test by overriding tearDown().

Each test runs in its own fixture so there can be no side effects among test runs.

Here is an example:

<?php
class MathTest extends PHPUnit_Framework_TestCase
{
    public $value1;
    public $value2;

    protected function setUp()
    {
        $this->value1 = 2;
        $this->value2 = 3;
    }
}
?>

For each test implement a method which interacts with the fixture. Verify the expected results with assertions specified by calling assert with a boolean.

<?php
public function testPass()
{
    $this->assertTrue($this->value1 + $this->value2 == 5);
}
?>
